박스모델

CSS에서 객체는 박스모델로 관리됩니다.
아래는 박스 모델을 간단히 그림으로 나타낸 것입니다. 이제 아래를 천천히 읽으며 이 그림의 의미를 이해해 봅시다!

박스모델의 구조

박스모델은 위 그림에서 보다시피 안에서부터 content, padding, border, margin 으로 나타낼 수 있습니다.
  1. content
    2. content은 객체에서 실제로 정보를 담고 있는 부분을 의미합니다.
  2. padding
    3. padding은 객체의 border와 content 사이의 공간을 의미합니다. 만약 border를 더 넓게 잡고 싶다면 padding을 증가시켜야 합니다.
  3. border
    4. border은 객체의 경계를 말합니다. border를 기준으로 객체의 안밖이 바뀝니다.
  4. margin
    5. margin은 border 바깥의 여백을 말합니다. 만약 서로 다른 두 객체의 border 사이의 거리를 더 넓게 잡고 싶다면 margin을 증가시켜야 합니다.

display 속성

display는 박스모델에서 각 객체의 content가 어떻게 표시될 지 결정하는 인자입니다. display에는 세 가지 종류의 값이 있습니다.
  1. inline
    2.  inline 값은 content가 한 줄의 일부가 되도록 합니다. 예를 들어 strong 태그는 inline으로 A와 같이 부분을 차지합니다.
  2. block
    3.  block 값은 content가 한 줄을 차지하도록 합니다. 예를 들어 제목 태그는 block으로 B와 같이 한 줄을 차지합니다.
  3. none
    4.  none 값은 content가 보이지 않게 합니다. 예를 들어 head 태그는 none으로 보이지 않습니다.

display와 태그

display와 관련된 태그도 있습니다. 이 display와 관련된 태그는 이 태그 안에 있는 부분의 display를 설정합니다.
  1. span
    2. span 태그는 안에 있는 내용이 inline display를 갖도록 합니다.
  2. div
    3. div 태그는 안에 있는 내용이 block display를 갖도록 합니다.
하지만 이는 태그의 특성으로, CSS의 display 속성을 조절하면 바꿀 수 있습니다.
또한 이 태그 2개는 display의 설정보다 내용을 묶는데 더 자주 사용되며 이는 중요한 성질입니다.

개발자 도구: 검사

모든 웹페이지에서 우클릭 후 '검사'를 누르면 이 웹페이지의 HTML 소스를 볼 수 있습니다.
우리가 흔히 개발자 도구라 부르는 이것은 박스모델을 이용한 개발에서 유용합니다.
개발자 도구가 유용한 이유는 HTML 소스에 커서를 가져다 놓았을 때, 그 소스가 만드는 객체를 박스모델의 형태로 스타일 탭에서 보여주기 때문입니다.
항상 개발 과정에서 디자인을 하다 보면 생기는 요구사항이 있습니다. 이때 개발자 도구를 이용해 어느 부분을 수정해야 할지 확인하시며 개발하시기 바랍니다.

문법

네 변 border 조절: border: (border 속성들)
한 변 border 조절: border-(left, right, top, bottom): (border 속성들)
border 속성들: border-width: ___px, border-color: (색) border-style: (선 종류)
네 변 padding 조절: padding: ___px"
한 변 padding 조절: padding-(left, right, top, bottom): ___px
네 변 margin 조절: margin: ___px
한 변 margin 조절: margin-(left, right, top, bottom): ___px
span 태그: <span>......</span>
div 태그: <div>......</div>
